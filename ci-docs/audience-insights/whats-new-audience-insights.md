# <a name="whats-new-in-the-audience-insights-capability-of-dynamics-365-customer-insights"></a>มีอะไรใหม่ในความสามารถข้อมูลเชิงลึกกลุ่มเป้าหมายของ Dynamics 365 Customer Insights



พวกเรารู้สึกตื่นเต้นที่จะประกาศการปรับปรุงใหม่ล่าสุดของเรา! บทความนี้สรุปคุณลักษณะตัวอย่างสำหรับสาธารณะ การปรับปรุงความพร้อมใช้งานทั่วไป และการปรับปรุงคุณลักษณะ เมื่อต้องการดูแผนคุณลักษะในระยะยาว ให้ดูที่ [แผนการนำออกใช้ของ Dynamics 365 และ Power Platform](/dynamics365/release-plans/)

เราเปิดตัวการปรับปรุงตามภูมิภาค ดังนั้น บางภูมิภาคอาจเห็นคุณลักษณะก่อนภูมิภาคอื่นๆ หากไม่ได้ระบุไว้แตกต่างกัน คุณไม่จำเป็นต้องดำเนินการใดๆ และเราจะปรับปรุงแอปโดยอัตโนมัติ โดยไม่ต้องหยุดทำงาน

> [!TIP]
> เมื่อต้องการส่งและโหวตคำขอคุณลักษณะและข้อเสนอแนะผลิตภัณฑ์ ให้ไปที่ [พอร์ทัลไอเดียเกี่ยวกับแอปพลิเคชัน Dynamics 365](https://experience.dynamics.com/ideas/categories/?forum=79a8c474-4e35-e911-a971-000d3a4f3343&forumName=Dynamics%20365%20Customer%20Insights)


## <a name="december-2021-updates"></a>การปรับปรุงประจำเดือนธันวาคม 2021

การอัปเดตในเดือนธันวาคม 2021 ประกอบด้วยคุณลักษณะใหม่ การอัปเกรดประสิทธิภาพ และการแก้ไขข้อบกพร่อง

### <a name="forward-customer-insights-logs-to-azure-monitor"></a>ส่งต่อบันทึก Customer Insights ไปยัง Azure Monitor

Customer Insights มีการรวมโดยตรงกับ Azure Monitor คุณลักษณะนี้รวมถึงเหตุการณ์การตรวจสอบและเหตุการณ์การดำเนินงาน บันทึกทรัพยากร Azure Monitor ให้คุณสามารถตรวจสอบและส่งบันทึกไปที่ ที่เก็บข้อมูล Azure, Azure Log Analytics หรือสตรีมไปที่ Azure Event Hubs

ดูข้อมูลเพิ่มเติมได้ที่ [การส่งต่อบันทึกใน Dynamics 365 Customer Insights ด้วย Azure Monitor (พรีวิว)](diagnostics.md)

### <a name="enrich-customer-profiles-with-engagement-data"></a>เพิ่มโปรไฟล์ลูกค้าด้วยข้อมูลการมีส่วนร่วม

ใช้ข้อมูลจาก Microsoft Office 365 เพื่อเพิ่มโปรไฟล์บัญชีลูกค้าของคุณด้วยข้อมูลเชิงลึกเกี่ยวกับการมีส่วนร่วมผ่านแอป Office 365 ข้อมูลการมีส่วนร่วมประกอบด้วยอีเมลและกิจกรรมการประชุมซึ่งรวบรวมไว้ที่ระดับบัญชี ตัวอย่างเช่น จำนวนอีเมลจากบัญชีธุรกิจหรือจำนวนการประชุมกับบัญชีนั้น ไม่มีการแชร์ข้อมูลเกี่ยวกับผู้ใช้แต่ละราย การเพิ่มความสมบูรณ์สามารถใช้งานเฉพาะในภูมิภาคต่อไปนี้: สหราชอาณาจักร ยุโรป เมริกาเหนือ

ดูข้อมูลเพิ่มเติมได้ที่ [เพิ่มโปรไฟล์ลูกค้าด้วยข้อมูลการมีส่วนร่วม (พรีวิว)](enrichment-office.md)

### <a name="advanced-data-unification-features"></a>คุณลักษณะการรวมข้อมูลขั้นสูง

#### <a name="enable-conflict-resolution-policies-at-the-individual-attribute-level"></a>เปิดใช้นโยบายการแก้ไขข้อขัดแย้งที่ระดับแอตทริบิวต์แต่ละรายการ

เมื่อขจัดข้อมูลซ้ำซ้อนของเรกคอร์ดลูกค้าภายในเอนทิตี คุณอาจไม่ต้องการเลือกเรกคอร์ดแบบเต็มในฐานะผู้ชนะ ตอนนี้เราอนุญาตให้คุณรวมฟิลด์ที่ดีที่สุดจากเรกคอร์ดต่างๆ ได้ตามกฎของทุกแอตทริบิวต์ ตัวอย่างเช่น คุณสามารถเลือกเก็บอีเมลล่าสุดและที่อยู่ที่สมบูรณ์ที่สุดจากเรกคอร์ดต่างๆ 

ขณะนี้ คุณสามารถกำหนดกฎการผสานที่แยกต่างหากสำหรับแอตทริบิวต์แต่ละรายการขณะที่ขจัดข้อมูลซ้ำซ้อนและผสานเรกคอร์ดภายในเอนทิตีเดียว ก่อนหน้านี้ เราให้คุณเลือกกฎการผสานเพียงกฎเดียวเท่านั้น (การเก็บเรกคอร์ดโดยยึดตามความสมบูรณ์ของข้อมูลใหม่) และกฎนั้นได้นำไปใช้กับแอตทริบิวต์ทั้งหมดในระดับเรกคอร์ด ซึ่งไม่เหมาะเมื่อพบข้อมูลบางส่วนที่คุณต้องการเก็บไว้ในเรกคอร์ด A และข้อมูลที่ดีอื่นๆ ที่พบในเรกคอร์ด B

สำหรับข้อมูลเพิ่มเติม โปรดดู [กำหนดการขจัดข้อมูลซ้ำซ้อนในเอนทิตีที่ตรงกัน](match-entities.md#define-deduplication-on-a-match-entity)

#### <a name="custom-rules-for-matching"></a>กฎที่กำหนดเองสำหรับการจับคู่

มีบางครั้งที่คุณต้องระบุข้อยกเว้นสำหรับกฎทั่วไปเพื่อไม่ให้จับคู่เรกคอร์ด กรณีนี้สามารถเกิดขึ้นเมื่อมีหลายคนแชร์ข้อมูลเพียงพอให้ระบบจับคู่ข้อมูลเป็นรายการเดียวกัน ตัวอย่างเช่น ฝาแฝดที่มีนามสกุล อาศัยอยู่ในเมืองเดียวกัน และมีวันเดือนปีเกิดเดียวกัน

ข้อยกเว้นช่วยให้แน่ใจว่าการรวมข้อมูลที่ไม่ถูกต้องสามารถแก้ไขได้ในกฎการรวม คุณสามารถเพิ่มข้อยกเว้นได้หลายข้อในกฎ

ดูข้อมูลเพิ่มเติมได้ที่ [เพิ่มข้อยกเว้นลงในกฎ](match-entities.md#add-exceptions-to-a-rule)

#### <a name="provide-additional-conflict-resolution-policies-and-enable-grouping-of-attributes"></a>ระบุนโยบายการแก้ไขข้อขัดแย้งเพิ่มเติมและเปิดใช้งานการจัดกลุ่มแอตทริบิวต์

คุณลักษณะนี้ช่วยให้คุณสามารถจัดกลุ่มของฟิลด์เป็นหน่วยเดียว ตัวอย่างเช่น เมื่อเรกคอร์ดของเราประกอบด้วยฟิลด์ ที่อยู่1, ที่อยู่2, เมือง รัฐ และรหัสไปรษณีย์ เราอาจไม่ต้องการผสานที่อยู่2 ของเรกคอร์ดอื่น โดยคิดว่าอาจจะทำให้ข้อมูลของเราสมบูรณ์ยิ่งขึ้น

ตอนนี้คุณสามารถรวมกลุ่มของฟิลด์ที่เกี่ยวข้องและนำนโยบายการผสานเดียวไปใช้กับกลุ่มได้แล้ว 

ดูข้อมูลเพิ่มเติมได้ที่ [รวมกลุ่มของฟิลด์](merge-entities.md#combine-a-group-of-fields)


## <a name="november-2021-updates"></a>การปรับปรุงประจำเดือนพฤศจิกายน 2021

การปรับปรุงในเดือนพฤศจิกายน 2021 ประกอบด้วยคุณลักษณะใหม่ การอัปเกรดประสิทธิภาพ และการแก้ไขข้อบกพร่อง

### <a name="segment-membership-now-available-in-dataverse"></a>สมาชิกเซ็กเมนต์สามารถใช้งานใน Dataverse ได้แล้ว

ข้อมูลสมาชิกเซ็กเมนต์สำหรับโปรไฟล์ลูกค้ามีอยู่ใน Dataverse พร้อมกับโปรไฟล์ลูกค้าและข้อมูลเชิงลึก แอปการดำเนินการของ Dynamics 365 และแอปแบบจำลองสามารถใช้ข้อมูลนี้เพื่อค้นหารายละเอียดของสมาชิกเซ็กเมนต์สำหรับลูกค้าที่กำหนด

### <a name="activities-support-contact-level-details-for-business-accounts"></a>กิจกรรมสนับสนุนรายละเอียดระดับผู้ติดต่อสำหรับบัญชีธุรกิจ

ตอนนี้คุณสามารถกำหนดค่า แสดง และกรองกิจกรรมสำหรับผู้ติดต่อในไทม์ไลน์กิจกรรมในบัญชีธุรกิจของคุณ เพื่อให้เข้าใจได้ดีขึ้นว่าผู้ติดต่อบัญชีใดมีส่วนร่วมในกิจกรรมเฉพาะ

## <a name="october-2021-updates"></a>การปรับปรุงประจำเดือนตุลาคม 2021

การปรับปรุงในเดือนตุลาคม 2021 ประกอบด้วยคุณลักษณะใหม่ การอัปเกรดประสิทธิภาพ และการแก้ไขข้อบกพร่อง

### <a name="b-to-b"></a>B-to-B

ตั้งแต่เดือนตุลาคม 2021 คุณสามารถทำงานกับบัญชีธุรกิจและผู้ติดต่อที่เกี่ยวข้องใน Customer Insights ก่อนหน้านี้ แอปส่วนใหญ่ได้รับการปรับแต่งให้เหมาะกับผู้บริโภคแต่ละราย พื้นที่คุณลักษณะต่างๆ ได้รับการปรับปรุงเพื่อรองรับสถานการณ์แบบ B-to-B นอกเหนือจากชนิดสภาพแวดล้อมใหม่ สำหรับภาพรวมเกี่ยวกับคุณลักษณะ B-to-B ที่รองรับ โปรดดูที่ [ทำงานกับบัญชีธุรกิจในข้อมูลเชิงลึกของผู้ชม](work-with-business-accounts.md)

ส่วนต่อไปนี้เน้นประเด็นสำคัญบางส่วนที่ได้รับการปรับเพื่อรองรับบัญชีธุรกิจและผู้บริโภคแต่ละราย

#### <a name="export-segments-based-on-business-accounts"></a>ส่งออกเซ็กเมนต์ตามบัญชีธุรกิจ

การส่งออกเซ็กเมนต์ทั้งหมดในข้อมูลเชิงลึกของผู้ชมมีอยู่ในบริบทของบัญชีธุรกิจ การส่งออกเซ็กเมนต์ส่วนใหญ่ต้องมีการกำหนดค่าเพิ่มเติมและ [ข้อมูลการติดต่อที่คาดการณ์ไว้](segment-builder.md#create-a-new-segment) ในเซ็กเมนต์พื้นฐานที่ถูกต้องสำหรับบัญชีธุรกิจ ดูข้อมูลเพิ่มเติมที่ [ส่งออกเซ็กเมนต์](export-destinations.md#export-segments)

#### <a name="use-the-linkedin-ads-export-with-business-accounts"></a>ใช้การส่งออกโฆษณา LinkedIn กับบัญชีธุรกิจ

การส่งออกโฆษณา LinkedIn พร้อมใช้งานแล้วสำหรับการติดต่อและการกำหนดเป้าหมายบริษัทในบริบทของบัญชีธุรกิจ เมื่อเลือกการกำหนดเป้าหมายบริษัทเป็นจุดสนใจหลักของการส่งออก LinkedIn คุณสามารถส่งออกเซ็กเมนต์ที่สร้างจากบัญชีธุรกิจโดยไม่ต้องแสดงข้อมูลการติดต่อ สำหรับข้อมูลเพิ่มเติม ไปที่เอกสารเกี่ยวกับ [การส่งออกโฆษณา LinkedIn](export-linkedin-ads.md) และความแตกต่างระหว่าง [กำหนดเป้าหมายการติดต่อ](https://business.linkedin.com/marketing-solutions/ad-targeting/contact-targeting) และ [กำหนดเป้าหมายบริษัท](https://business.linkedin.com/marketing-solutions/ad-targeting/account-targeting) 

#### <a name="create-measures-based-on-business-accounts-and-their-hierarchy"></a>สร้างการวัดตามบัญชีธุรกิจและลำดับชั้นของบัญชี

ตัวสร้างการวัดช่วยให้คุณสร้างการวัดสำหรับบัญชีธุรกิจและใช้ข้อมูลลำดับชั้นได้ ข้อมูลลำดับชั้นใช้เพื่อรวบรวมการคำนวณการวัดในบัญชีและบัญชีย่อยที่เกี่ยวข้องทั้งหมด ตัวอย่างเช่น คุณสามารถสร้างการวัด เช่น รายได้รวมสำหรับบัญชีธุรกิจแต่ละกลุ่มที่ระบุโดยลำดับชั้น ดูข้อมูลเพิ่มเติมที่ [กำหนดและจัดการการวัด](measures.md)

#### <a name="create-segments-based-on-business-accounts-and-their-hierarchy"></a>สร้างเซ็กเมนต์ตามบัญชีธุรกิจและลำดับชั้นของบัญชี

ตัวสร้างเซ็กเมนต์ช่วยให้คุณสามารถสร้างเซ็กเมนต์ของบัญชีธุรกิจที่รวมข้อมูลการติดต่อสำหรับแต่ละบัญชีในเซ็กเมนต์ได้ หากคุณมีการตั้งค่าลำดับชั้นของบัญชี คุณสามารถใช้ข้อมูลลำดับชั้นของบัญชีในการสร้างเซ็กเมนต์ ดูข้อมูลเพิ่มเติมที่ [สร้างเซ็กเมนต์ใหม่](segment-builder.md#create-a-new-segment)

#### <a name="retain-your-business-accounts-with-deep-insights-to-their-churn-tendency"></a>รักษาบัญชีธุรกิจของคุณด้วยข้อมูลเชิงลึกเกี่ยวกับแนวโน้มการเลิกใช้บริการ

ตอนนี้รูปแบบการคาดคะเนการเลิกใช้บริการของลูกค้ารองรับบัญชีธุรกิจด้วย คุณสามารถประเมินความเสี่ยงของการเลิกใช้ ไม่ใช่แค่สำหรับบัญชีเท่านั้น แต่ยังรวมถึงการรวมกันของบัญชีและประเภทผลิตภัณฑ์หรือบริการที่ลูกค้าซื้อจากคุณด้วย การเพิ่มเติมนี้ช่วยให้คุณเข้าใจว่าโดยทั่วไปแล้วบัญชีมีแนวโน้มมากขึ้นที่จะหยุดซื้อจากคุณหรือเฉพาะสินค้าหรือบริการบางประเภท เพื่อช่วยให้คุณใช้โมเดล AI นี้เพิ่มเติมได้ ยังมีการแสดงสาเหตุที่บัญชีมีแนวโน้มที่จะเลิกใช้ด้วย ดูข้อมูลเพิ่มเติมที่ [การคาดคะเนการเลิกทำธุรกรรม (พรีวิว)](predict-transactional-churn.md)

## <a name="april-2021-updates"></a>ปรับปรุง เมษายน 2021

การปรับปรุงในเดือนเมษายน 2021 ประกอบด้วยคุณลักษณะ การอัปเกรดประสิทธิภาพ และการแก้ไขข้อบกพร่องต่างๆ

### <a name="data-unification"></a>การรวมข้อมูล
 
- **ประสบการณ์การผสานขั้นสูงสำหรับการรวมข้อมูล**    
  
   ขณะนี้เรามีประสบการณ์ของผู้ใช้ขั้นสูงในการกำหนดค่าการผสานของกระบวนการรวมข้อมูล การเปลี่ยนแปลงรวมถึงการจัดลำดับที่ใช้งานง่ายของฟิลด์ที่ผสานและสถิติโดยละเอียดเกี่ยวกับฟิลด์แบบรวมและแบบเดี่ยว

- **การจัดลำดับเอนทิตีใหม่และกำหนดค่าเรกคอร์ดแหล่งที่มาทั้งหมดลงในเอนทิตีลูกค้า**  
      
   ตอนนี้คุณสามารถจัดลำดับใหม่และลบเอนทิตีออกจากแผนการรวมกลุ่มที่มีอยู่ในกระบวนการรวมข้อมูลได้ ซึ่งให้ความยืดหยุ่นในการจัดลำดับเอนทิตีใหม่ในกระบวนการจับคู่ตามความต้องการทางธุรกิจ นอกจากนี้ เรายังเปิดใช้งานรวมเรกคอร์ดที่ไม่ตรงกันทั้งหมดไว้ในเอนทิตี *ลูกค้า* สุดท้าย ซึ่งช่วยให้พวกเขาสามารถกำหนดนิยามชุดข้อมูลโปรไฟล์ลูกค้าของพวกเขาได้

### <a name="enrichments"></a>การเพิ่มข้อมูล

 - **การเพิ่มข้อมูลใหม่: ที่อยู่ที่ปรับปรุงแล้ว**    
  
   เรามีความยินดีที่จะแนะนำการเพิ่มข้อมูลใหม่เพื่อเพิ่มความสมบูรณ์ที่อยู่ในข้อมูลลูกค้าของคุณ ที่อยู่ในข้อมูลของคุณอาจไม่มีโครงสร้าง ไม่สมบูรณ์ หรือไม่ถูกต้อง คุณลักษณะนี้ใช้โมเดลของ Microsoft เพื่อกำหนดมาตรฐานและเพิ่มความสมบูรณ์ที่อยู่ของคุณลงในรูปแบบ Common Data Model เพื่อความแม่นยำและข้อมูลเชิงลึกที่ดีขึ้น
 
   ดูข้อมูลเพิ่มเติมที่ [การเพิ่มข้อมูลของโปรไฟล์ลูกค้าด้วยที่อยู่ที่ปรับปรุงแล้ว](enrichment-enhanced-addresses.md)

- **ประสบการณ์การกำหนดค่าที่แนะนำสำหรับการเพิ่มข้อมูล**    
  
   เราได้ทบทวนประสบการณ์การกำหนดค่าสำหรับการเพิ่มข้อมูลด้วยประสบการณ์ที่เรียบง่ายและที่แนะนำ ตอนนี้คุณมีกระบวนการที่ชัดเจนทีละขั้นตอนสำหรับการสร้างและการแก้ไขการเพิ่มข้อมูล
 
   นอกจากนี้ เรายังแยกการกำหนดค่าของการเชื่อมต่อสำหรับการเพิ่มข้อมูลของบุคคลที่สาม เพื่อให้สามารถใช้การเชื่อมต่อเดียวกันโดยการเพิ่มข้อมูลหลายรายการ เฉพาะผู้ดูแลระบบเท่านั้นที่สามารถกำหนดค่าการเชื่อมต่อใหม่ แต่การเชื่อมต่อที่สร้างขึ้นจะพร้อมใช้งานสำหรับทั้งผู้ดูแลระบบและผู้สนับสนุน    

   สำหรับข้อมูลเพิ่มเติม ดู [ภาพรวมของการเชื่อมต่อ](connections.md)

- **การเพิ่มข้อมูลหลายรายการในชนิดเดียวกัน**    
  
   ตอนนี้เราอนุญาตให้ผู้ใช้สร้างและจัดการการเพิ่มข้อมูลหลายรายการในชนิดการเพิ่มข้อมูลเดียวกัน ตัวอย่างเช่น ตอนนี้คุณสามารถสร้างการเพิ่มข้อมูลที่อยู่สองรายการแยกกัน เพื่อการเพิ่มข้อมูลเซ็กเมนต์ลูกค้าที่แตกต่างกันสองเซ็กเมนต์ ข้อจำกัดมีผลกับจำนวนการเพิ่มข้อมูลชนิดเดียวกันที่สามารถสร้างได้ และแตกต่างกันไปโดยขึ้นอยู่กับชนิดการเพิ่มข้อมูล
  
   ดูข้อมูลเพิ่มเติมที่ [การเพิ่มข้อมูลสำหรับโปรไฟล์ลูกค้า](enrichment-hub.md)

## <a name="march-2021-updates"></a>การปรับปรุงเดือนมีนาคม 2021

การปรับปรุงในเดือนมีนาคม 2021 ประกอบด้วยคุณสมบัติต่าง ๆ การอัปเกรดประสิทธิภาพ และการแก้ไขข้อบกพร่อง

### <a name="activities"></a>กิจกรรม

- **ตัวช่วยสร้างกิจกรรมและชนิดความหมาย**

   เราได้ปรับปรุงและปรับปรุงประสบการณ์การแมปกิจกรรมของเรา เพื่อเป็นแนวทางและลดความซับซ้อนในการสร้างการแมปกิจกรรม ในประสบการณ์ใหม่นี้ ผู้ใช้จะได้รับประสบการณ์ที่แนะนำเพื่อช่วยทำแต่ละขั้นตอนของกระบวนการให้เสร็จสิ้น ในขั้นตอนการแมปกิจกรรม นอกเหนือจากการเลือกชนิดกิจกรรมต่าง ๆ แล้ว ผู้ใช้ยังสามารถเลือกที่จะแมปข้อมูลสำหรับความหมายได้อีกด้วยสำหรับ *การสมัครสมาชิก* และ/หรือ *SalesOrderLine* เป็นสคีมามาตรฐานอุตสาหกรรม ซึ่งสามารถใช้สำหรับการบริโภคดาวน์สตรีม   

   สำหรับข้อมูลเพิ่มเติม ดู [กิจกรรมของลูกค้า](activities.md)

### <a name="data-ingestion"></a>การนำเข้าข้อมูล

- **เชื่อมต่อกับแหล่งข้อมูลภายในองค์กรโดยใช้โฟลว์ข้อมูลและเกตเวย์ Power Platform** เรายินดีที่จะประกาศตัวอย่างของโฟลว์ข้อมูล Power Platform และการเชื่อมต่อภายในองค์กรโดยใช้เกตเวย์ใน Customer Insights ที่เชื่อมโยงสภาพแวะล้อม Power Platform หรือ Dataverse แหล่งข้อมูลใหม่ใด ๆ ที่สร้างขึ้นในสภาพแวดล้อม Customer Insights ที่มีการเชื่อมโยงสภาพแวดล้อม Dataverse จะเริ่มต้นเป็นโฟลว์ข้อมูล Power Platform ที่นำการเชื่อมต่อข้อมูลภานในองค์กรและชุดตัวเชื่อมต่อและความสามารถในการแปลง

### <a name="extensibility"></a>ความสามารถในการเพิ่ม

- **การส่งออกที่จัดระเบียบในการเชื่อมต่อและการส่งออก** เราได้เปลี่ยนชื่อของหน้า **ปลายทางการส่งออก** เป็น **การเชื่อมต่อ** และเพิ่มหน้าแยกสำหรับ **การส่งออก** ในการปรับปรุงนี้ เราจะเปลี่ยนการส่งออกที่มีอยู่เป็นคู่ของการเชื่อมต่อและการส่งออกโดยใช้การเชื่อมต่อนั้น ขณะนี้ผู้ดูแลระบบมีความชัดเจนมากขึ้นเกี่ยวกับข้อมูลขาออกในหน้า **การเชื่อมต่อ** บทบาทของผู้ใช้ทั้งหมดสามารถเข้าถึงหน้า **การส่งออก** แต่เฉพาะผู้ดูแลระบบเท่านั้นที่สามารถเลือกที่จะอนุญาตให้ผู้สนับสนุนแก้ไขการส่งออกเฉพาะด้วยการเชื่อมต่อที่แบ่งปันได้     
  ดูข้อมูลเพิ่มเติมที่ [ภาพรวมการเชื่อมต่อ](connections.md) และ [ภาพรวมการส่งออก](export-destinations.md)

- **ส่งออกเซ็กเมนต์ไปยัง Campaign Monitor** เราได้ขยายปลายทางการส่งออกของเราเพื่อรวม Campaign Monitor ตอนนี้คุณสามารถส่งออกเซ็กเมนต์จาก Customer Insights ไปยังรายการ Campaign Monitor และใช้เป็นพื้นฐานสำหรับแคมเปญการตลาดของคุณได้    
   ดูข้อมูลเพิ่มเติมที่ [ส่งออกไปยัง Campaign Monitor](export-campaign-monitor.md)

- **ส่งออกเซ็กเมนต์ไปยัง Constant Contact** เราได้ขยายปลายทางการส่งออกของเราเพื่อรวม Constant Contact ตอนนี้คุณสามารถส่งออกเซ็กเมนต์จาก Customer Insights ไปยังรายการ Constant Contact และใช้เป็นพื้นฐานสำหรับแคมเปญการตลาดของคุณได้   
   ดูข้อมูลเพิ่มเติมที่ [ส่งออกไปยัง Constant Contact](export-constant-contact.md)

- **ส่งออกเซ็กเมนต์ไปยัง RollWorks** เราได้ขยายปลายทางการส่งออกของเราเพื่อรวม RollWorks ตอนนี้คุณสามารถส่งออกเซ็กเมนต์จาก Customer Insights ไปยังผู้ชม RollWorks และใช้เป็นพื้นฐานสำหรับการโฆษณาแบบ B-to-B ของคุณได้แล้ว    
   ดูข้อมูลเพิ่มเติมที่ [ส่งออกข้อมูลไปยัง RollWorks ](export-rollworks.md)

- **ส่งออกเซ็กเมนต์ไปยัง Snapchat** เราได้ขยายปลายทางการส่งออกของเราเพื่อรวม Snapchat ตอนนี้คุณสามารถส่งออกเซ็กเมนต์จาก Customer Insights ไปยังผู้ชมบน Snapchat และใช้เป็นพื้นฐานสำหรับโฆษณาของคุณได้     
   ดูข้อมูลเพิ่มเติมที่ [ส่งออกข้อมูลไปยัง Snapchat](export-snapchat.md)

### <a name="predictions"></a>การคาดคะเน

- **ใช้ตัวกรองผลิตภัณฑ์ในคำแนะนำผลิตภัณฑ์เชิงคาดคะเน** เราได้เพิ่มความสามารถในการใช้ตัวกรองผลิตภัณฑ์ในรูปแบบการแนะนำผลิตภัณฑ์ของเรา ตอนนี้คุณสามารถสร้างการคาดคะเนที่ใช้เฉพาะส่วนย่อยของผลิตภัณฑ์ของคุณ    
   ดูข้อมูลเพิ่มเติมที่ [กำหนดค่าตัวกรองผลิตภัณฑ์](predict-product-recommendation.md#configure-product-filters)

- **สร้างเซ็กเมนต์จากการคาดคะเนแบบจำลอง** เราได้เพิ่มวิธีที่รวดเร็วในการสร้างเซ็กเมนต์โดยใช้ผลลัพธ์ของแบบจำลองการคาดคะเน จากหน้าผลลัพธ์ของแบบจำลอง คุณสามารถสร้างเซ็กเมนต์ใหม่ได้อย่างง่ายดายโดยเลือกตัวเลือก **สร้างเซ็กเมนต์** ใหม่    
  ดูข้อมูลเพิ่มเติมที่ [สร้างเซ็กเมนต์ตามแบบจำลองการคาดคะเน](prediction-based-segment.md)

- **คำอธิบายสำหรับคำแนะนำผลิตภัณฑ์** เราได้เพิ่มข้อมูลที่อธิบายถึงปัจจัยสำคัญที่ได้เรียนรู้จากโมเดล AI เพื่อสร้างคำแนะนำผลิตภัณฑ์และระดับที่ปัจจัยเหล่านั้นมีส่วนต่อคำแนะนำผลิตภัณฑ์ ข้อมูลนี้จะถูกเพิ่มลงในหน้าจอผลลัพธ์ของแบบจำลอง    
   ดูข้อมูลเพิ่มเติมที่ [ตรวจสอบสถานะการคาดคะเนและผลลัพธ์](predict-product-recommendation.md#review-a-prediction-status-and-results)

## <a name="february-2021-updates"></a>การปรับปรุงเดือนกุมภาพันธ์ 2021

การปรับปรุงในเดือนกุมภาพันธ์ 2021 ประกอบด้วยคุณลักษณะต่างๆ การอัปเกรดประสิทธิภาพ และการแก้ไขข้อบกพร่อง

#### <a name="extensibility"></a>ความสามารถในการเพิ่ม

- **ส่งออกเซ็กเมนต์ไปยัง AdRoll**

  เราได้ขยายปลายทางการส่งออกของเราเพื่อรวม AdRoll ตอนนี้คุณสามารถส่งออกเซ็กเมนต์จาก Customer Insights ไปยังกลุ่มเป้าหมาย AdRoll และใช้เป็นข้อมูลพื้นฐานสำหรับการโฆษณาของคุณได้ ดูข้อมูลเพิ่มเติมที่ [ตัวเชื่อมต่อสำหรับ AdRoll](export-adroll.md)

#### <a name="segments"></a>เซ็กเมนต์
 
- **ทำซ้ำเซ็กเมนต์**
  
  หากต้องการสร้างเซ็กเมนต์ใหม่ตามเซ็กเมนต์ที่มีอยู่ ตอนนี้คุณสามารถทำซ้ำเซ็กเมนต์และแก้ไขเซ็กเมนต์ที่ซ้ำกันเพื่อปรับปรุงเพิ่มเติมได้ 

- **เพิ่มแอตทริบิวต์เพิ่มเติมให้กับเซ็กเมนต์**

  ตอนนี้ คุณสามารถรวมแอตทริบิวต์ในผลลัพธ์ของเซ็กเมนต์ของคุณได้ แม้ว่าแอตทริบิวต์เหล่านี้จะไม่ได้เป็นส่วนหนึ่งของโปรไฟล์ลูกค้าก็ตาม ตัวอย่างเช่น รวมรหัสการสมัครสมาชิกในเซ็กเมนต์แม้ว่าจะเป็นส่วนหนึ่งของเอนทิตีการสมัครสมาชิกที่มีความสัมพันธ์แบบ M:1 กับเอนทิตีลูกค้า ตราบเท่าที่แอตทริบิวต์เป็นของเอนทิตีที่เกี่ยวข้องกับเอนทิตีลูกค้า คุณสามารถรวมแอตทริบิวต์เหล่านี้ได้  

#### <a name="predictions"></a>การคาดคะเน

- **สร้างคำแนะนำผลิตภัณฑ์ที่คาดคะเนได้**

  การทำความเข้าใจว่าลูกค้าสนใจซื้ออะไรเป็นขั้นตอนแรกที่จำเป็นในการปรับปรุงรายได้ของธุรกิจและสร้างความภักดีของลูกค้าผ่านการปรับเปลี่ยนตามแบบเฉพาะและการมีส่วนร่วม การให้คำแนะนำสำหรับผลิตภัณฑ์ที่ไม่สอดคล้องกับความสนใจของลูกค้าสามารถสร้างความรู้สึกตัดสัมพันธ์ระหว่างลูกค้ากับธุรกิจของคุณ และในที่สุดก็จำกัดรายได้และประสบการณ์โดยรวมที่เป็นไปได้สำหรับลูกค้า 

  ด้วยการใช้ข้อมูลของคุณเอง ตอนนี้คุณสามารถสร้างการคาดคะเนสำหรับผลิตภัณฑ์ที่ลูกค้าของคุณมีแนวโน้มที่จะซื้อในอนาคตได้แล้ว ดูข้อมูลเพิ่มเติมที่ [การคาดคะเนคำแนะนำผลิตภัณฑ์](predict-product-recommendation.md)

#### <a name="system-administration"></a>ผู้ดูแลระบบ

- **การคัดลอกสภาพแวดล้อมรองรับแหล่งข้อมูลประเภทต่างๆ มากขึ้น**

  ผู้ดูแลระบบสามารถคัดลอกการกำหนดค่าสภาพแวดล้อมไปยังสภาพแวดล้อมใหม่ในองค์กรเดียวกัน คุณลักษณะนี้ขยายฟังก์ชันการทำงานคัดลอกสภาพแวดล้อมสำหรับกรณีที่แหล่งข้อมูลขึ้นอยู่กับที่จัดเก็บข้อมูลดิบที่มีการจัดการ Microsoft Dataverse หรือมีการใช้โฟลเดอร์ Common Data Model

## <a name="january-2021-updates"></a>การปรับปรุงประจำเดือนมกราคม 2021

การปรับปรุงในเดือนมกราคม 2021 ประกอบด้วยคุณลักษณะต่างๆ การอัปเกรดประสิทธิภาพ และการแก้ไขข้อบกพร่อง

#### <a name="extensibility"></a>ความสามารถในการเพิ่ม

- **ฟังก์ชันเพิ่มเติมและประสิทธิภาพที่เพิ่มขึ้นสำหรับการส่งออก SFTP** ตอนนี้คุณสามารถส่งออกเอนทิตีผลลัพธ์ทั้งหมดจาก Customer Insights ไปยังโฮสต์ SFTP ได้ ก่อนหน้านี้ การส่งออกถูกจำกัดไว้ที่เอนทิตีเซ็กเมนต์ นอกจากนี้ ประสิทธิภาพของการส่งออก SFTP ยังช่วยให้ปริมาณข้อมูลมากขึ้นโดยใช้เวลาน้อยลงโดยขึ้นอยู่กับประสิทธิภาพของโฮสต์ SFTP ของคุณ    
  ดูข้อมูลเพิ่มเติมที่ [ตัวเชื่อมต่อสำหรับ SFTP (พรีวิว)](export-sftp.md)  

#### <a name="segments"></a>เซ็กเมนต์

- **การเรียนรู้เกี่ยวกับเครื่องที่ดำเนินการโดยเซ็กเมนต์ที่แนะนําเพื่อปรับปรุงเมตริก** มีวิธีใหม่ในการค้นหาและสร้างเซ็กเมนต์ ระบบใช้โมเดล AI เพื่อแนะนำเซ็กเมนต์ที่สามารถช่วยปรับปรุง KPI (การวัดผล) ที่คุณกำลังติดตามอยู่ เราแสดงขอบเขตของอิทธิพลของแอตทริบิวต์ที่คุณเลือกในการวัดหรือแอตทริบิวต์หลักอื่น ๆ ข้อมูลนี้ช่วยในการค้นหาเซ็กเมนต์ที่เป็นไปได้ที่นำเสนอโอกาส    
  ดูข้อมูลเพิ่มเติมที่ [เซ็กเมนต์แนะนํา (พรีวิว)](suggested-segments.md)

#### <a name="data-unification"></a>การรวมข้อมูล

- **ประสบการณ์การจับคู่ที่เพิ่มขึ้น** ในพื้นที่การรวมข้อมูล ประสบการณ์การจับคู่ได้รับการปรับปรุง ช่วยให้คุณกำหนดค่าและดูกฎการจับคู่ รวมถึงสถิติโดยละเอียดเพื่ออธิบายเพิ่มเติมว่าการจับคู่ทำงานอย่างไร มีตัวเลือกในการปิดใช้กฎการจับคู่ ดังนั้นจึงไม่ทำงานอีกต่อไป ในขณะที่ยังคงการกำหนดค่า ลากและวางกฎการจับคู่ และอื่น ๆ
  ดูข้อมูลเพิ่มเติมที่ [จับคู่เอนทิตี](match-entities.md)

- **ผลลัพธ์การลบข้อมูลซ้ำซ้อนจากกระบวนการจับคู่พร้อมใช้งานในรูปแบบเอนทิตี** ขณะนี้ผลลัพธ์ของกระบวนการคัดลอกข้อมูลซ้ำซ้อนจากกระบวนการจับคู่ถูกเขียนลงในเอนทิตีแยกต่างหากสำหรับการวิเคราะห์เพิ่มเติม เอนทิตีนี้ประกอบด้วยฟิลด์ที่ใช้ในกระบวนการขจัดข้อมูลซ้ำซ้อนและเรกคอร์ดผู้ชนะและเรกคอร์ดสำรองที่เกี่ยวข้องซึ่งรวมเข้ากับเรกคอร์ดผู้ชนะ
  ดูข้อมูลเพิ่มเติมที่ [ผลลัพธ์การขจัดข้อมูลซ้ำซ้อนเป็นเอนทิตี](match-entities.md#deduplication-output-as-an-entity)

#### <a name="system-administration"></a>ผู้ดูแลระบบ

- **แบ่งปันข้อมูลไปยัง Microsoft Dataverse อย่างราบรื่น** ตอนนี้คุณสามารถแชร์ผลลัพธ์ Customer Insights กับแอปพลิเคชัน Microsoft Dataverse โดยใช้ที่จัดเก็บข้อมูลดิบที่มีการจัดการ Microsoft Dataverse เมื่อคุณเชื่อมโยงสภาพแวดล้อม Dataverse ด้วย Customer Insights คุณจะได้รับตัวเลือกในการเปิดใช้งานการแบ่งปันข้อมูล
  สำหรับข้อมูลเพิ่มเติม ดู [จัดการการสภาพแวดล้อม](manage-environments.md)
