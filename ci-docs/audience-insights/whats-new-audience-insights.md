# <a name="whats-new-in-the-audience-insights-capability-of-dynamics-365-customer-insights"></a>มีอะไรใหม่ในความสามารถข้อมูลเชิงลึกกลุ่มเป้าหมายของ Dynamics 365 Customer Insights

[!INCLUDE [cc-data-platform-banner](../includes/cc-data-platform-banner.md)]

พวกเรารู้สึกตื่นเต้นที่จะประกาศการปรับปรุงใหม่ล่าสุดของเรา! บทความนี้สรุปคุณลักษณะตัวอย่างสำหรับสาธารณะ การปรับปรุงความพร้อมใช้งานทั่วไป และการอัปเดตคุณลักษณะ เมื่อต้องการดูแผนคุณลักษะในระยะยาว ให้ดูที่ [แผนการนำออกใช้ของ Dynamics 365 และ Power Platform](/dynamics365/release-plans/)

นอกจากนี้คุณยังสามารถดูวิดีโอต่อไปนี้เพื่อเรียนรู้เพิ่มเติมเกี่ยวกับความสามารถที่วางแผนไว้ในช่วงหกเดือนที่ผ่านมา

> [!VIDEO https://www.youtube.com/embed/jQh-7pscH30]

เราเปิดตัวการอัปเดตตามภูมิภาค ดังนั้น บางภูมิภาคอาจเห็นคุณลักษณะก่อนภูมิภาคอื่นๆ หากไม่ได้ระบุไว้แตกต่างกัน คุณไม่จำเป็นต้องดำเนินการใดๆ และเราจะอัปเดตแอปโดยอัตโนมัติ โดยไม่ต้องหยุดทำงาน

> [!TIP]
> เมื่อต้องการ ส่งและลงคะแนนให้กับ การร้องขอคุณลักษณะ และ คำแนะนำผลิตภัณฑ์ ไปที่ [พอร์ทัลแนวคิดเกี่ยวกับโปรแกรมประยุกต์ Dynamics 365](https://experience.dynamics.com/ideas/categories/?forum=79a8c474-4e35-e911-a971-000d3a4f3343&forumName=Dynamics%20365%20Customer%20Insights)

## <a name="february-2021-updates"></a>การปรับปรุงเดือนกุมภาพันธ์ 2021

การปรับปรุงในเดือนกุมภาพันธ์ 2021 ประกอบด้วยคุณลักษณะต่างๆ การอัปเกรดประสิทธิภาพ และการแก้ไขข้อบกพร่อง

#### <a name="extensibility"></a>ความสามารถในการเพิ่ม

- **ส่งออกเซ็กเมนต์ไปยัง AdRoll**

  เราได้ขยายปลายทางการส่งออกของเราเพื่อรวม AdRoll ตอนนี้คุณสามารถส่งออกเซ็กเมนต์จาก Customer Insights ไปยังกลุ่มเป้าหมาย AdRoll และใช้เป็นข้อมูลพื้นฐานสำหรับการโฆษณาของคุณได้ สำหรับข้อมูลเพิ่มเติม โปรดดู [ตัวเชื่อมต่อสำหรับ AdRoll](export-adroll.md)

#### <a name="segments"></a>เซ็กเมนต์
 
- **ทำซ้ำเซ็กเมนต์**
  
  หากต้องการสร้างเซ็กเมนต์ใหม่ตามเซ็กเมนต์ที่มีอยู่ ตอนนี้คุณสามารถทำซ้ำเซ็กเมนต์และแก้ไขเซ็กเมนต์ที่ซ้ำกันเพื่อปรับปรุงเพิ่มเติมได้ 

- **เพิ่มแอตทริบิวต์เพิ่มเติมให้กับเซ็กเมนต์**

  ตอนนี้ คุณสามารถรวมแอตทริบิวต์ในผลลัพธ์ของเซ็กเมนต์ของคุณได้ แม้ว่าแอตทริบิวต์เหล่านี้จะไม่ได้เป็นส่วนหนึ่งของโปรไฟล์ลูกค้าก็ตาม ตัวอย่างเช่น รวมรหัสการสมัครสมาชิกในเซ็กเมนต์แม้ว่าจะเป็นส่วนหนึ่งของเอนทิตีการสมัครสมาชิกที่มีความสัมพันธ์แบบ M:1 กับเอนทิตีลูกค้า ตราบเท่าที่แอตทริบิวต์เป็นของเอนทิตีที่เกี่ยวข้องกับเอนทิตีลูกค้า คุณสามารถรวมแอตทริบิวต์เหล่านี้ได้  

#### <a name="predictions"></a>การคาดคะเน

- **สร้างคำแนะนำผลิตภัณฑ์ที่คาดคะเนได้**

  การทำความเข้าใจว่าลูกค้าสนใจซื้ออะไรเป็นขั้นตอนแรกที่จำเป็นในการปรับปรุงรายได้ของธุรกิจและสร้างความภักดีของลูกค้าผ่านการปรับเปลี่ยนตามแบบเฉพาะและการมีส่วนร่วม การให้คำแนะนำสำหรับผลิตภัณฑ์ที่ไม่สอดคล้องกับความสนใจของลูกค้าสามารถสร้างความรู้สึกตัดสัมพันธ์ระหว่างลูกค้ากับธุรกิจของคุณ และในที่สุดก็จำกัดรายได้และประสบการณ์โดยรวมที่เป็นไปได้สำหรับลูกค้า 

  ด้วยการใช้ข้อมูลของคุณเอง ตอนนี้คุณสามารถสร้างการคาดคะเนสำหรับผลิตภัณฑ์ที่ลูกค้าของคุณมีแนวโน้มที่จะซื้อในอนาคตได้แล้ว สำหรับข้อมูลเพิ่มเติม โปรดดู [การคาดคะเนคำแนะนำผลิตภัณฑ์](predict-product-recommendation.md)

#### <a name="system-administration"></a>ผู้ดูแลระบบ

- **การคัดลอกสภาพแวดล้อมรองรับแหล่งข้อมูลประเภทต่างๆ มากขึ้น**

  ผู้ดูแลระบบสามารถคัดลอกการกำหนดค่าสภาพแวดล้อมไปยังสภาพแวดล้อมใหม่ในองค์กรเดียวกัน คุณลักษณะนี้จะขยายฟังก์ชันการทำงานของการคัดลอกสภาพแวดล้อมสำหรับกรณีที่แหล่งข้อมูลอิงอยู่กับที่จัดเก็บข้อมูลดิบ Common Data Service หรือมีการใช้โฟลเดอร์ Common Data Model

## <a name="january-2021-updates"></a>การอัปเดตประจำเดือนมกราคม 2021

การปรับปรุงในเดือนมกราคม 2021 ประกอบด้วยคุณลักษณะต่างๆ การอัปเกรดประสิทธิภาพ และการแก้ไขข้อบกพร่อง

#### <a name="extensibility"></a>ความสามารถในการเพิ่ม

- **ฟังก์ชันเพิ่มเติมและประสิทธิภาพที่เพิ่มขึ้นสำหรับการส่งออก SFTP** ตอนนี้คุณสามารถส่งออกเอนทิตีผลลัพธ์ทั้งหมดจาก Customer Insights ไปยังโฮสต์ SFTP ได้ ก่อนหน้านี้ การส่งออกถูกจำกัดไว้ที่เอนทิตีเซ็กเมนต์ นอกจากนี้ ประสิทธิภาพของการส่งออก SFTP ยังช่วยให้ปริมาณข้อมูลมากขึ้นโดยใช้เวลาน้อยลงโดยขึ้นอยู่กับประสิทธิภาพของโฮสต์ SFTP ของคุณ    
  สำหรับข้อมูลเพิ่มเติม โปรดดู [ตัวเชื่อมต่อสำหรับ SFTP (ตัวอย่าง)](export-sftp.md)  

#### <a name="segments"></a>เซ็กเมนต์

- **การเรียนรู้เกี่ยวกับเครื่องที่ดำเนินการโดยเซ็กเมนต์ที่แนะนําเพื่อปรับปรุงเมตริก** มีวิธีใหม่ในการค้นหาและสร้างเซ็กเมนต์ ระบบใช้แบบจำลอง AI เพื่อแนะนำเซ็กเมนต์ที่สามารถช่วยปรับปรุง KPI (การวัดผล) ที่คุณกำลังติดตามอยู่ เราแสดงขอบเขตของอิทธิพลของแอตทริบิวต์ที่คุณเลือกในการวัดหรือแอตทริบิวต์หลักอื่น ๆ ข้อมูลนี้ช่วยในการค้นหาเซ็กเมนต์ที่เป็นไปได้ที่นำเสนอโอกาส    
  สำหรับข้อมูลเพิ่มเติม โปรดดู [เซ็กเมนต์แนะนํา (ตัวอย่าง)](suggested-segments.md)

#### <a name="data-unification"></a>การรวมข้อมูล

- **ประสบการณ์การจับคู่ที่เพิ่มขึ้น** ในพื้นที่การรวมข้อมูล ประสบการณ์การจับคู่ได้รับการอัปเดต ช่วยให้คุณกำหนดค่าและดูกฎการจับคู่ รวมถึงสถิติโดยละเอียดเพื่ออธิบายเพิ่มเติมว่าการจับคู่ทำงานอย่างไร มีตัวเลือกในการปิดใช้กฎการจับคู่ ดังนั้นจึงไม่ทำงานอีกต่อไป ในขณะที่ยังคงการกำหนดค่า ลากและวางกฎการจับคู่ และอื่น ๆ
  สำหรับข้อมูลเพิ่มเติม โปรดดูที่ [จับคู่เอนทิตี](match-entities.md)

- **ผลลัพธ์การลบข้อมูลซ้ำซ้อนจากกระบวนการจับคู่พร้อมใช้งานในรูปแบบเอนทิตี** ขณะนี้ผลลัพธ์ของกระบวนการคัดลอกข้อมูลซ้ำซ้อนจากกระบวนการจับคู่ถูกเขียนลงในเอนทิตีแยกต่างหากสำหรับการวิเคราะห์เพิ่มเติม เอนทิตีนี้ประกอบด้วยฟิลด์ที่ใช้ในกระบวนการขจัดข้อมูลซ้ำซ้อนและเรกคอร์ดผู้ชนะและเรกคอร์ดสำรองที่เกี่ยวข้องซึ่งรวมเข้ากับเรกคอร์ดผู้ชนะ
  สำหรับข้อมูลเพิ่มเติม โปรดดู [ผลลัพธ์การขจัดข้อมูลซ้ำซ้อนเป็นเอนทิตี](match-entities.md#deduplication-output-as-an-entity)

#### <a name="system-administration"></a>ผู้ดูแลระบบ

- **แบ่งปันข้อมูลไปยัง Microsoft Dataverse อย่างราบรื่น** ตอนนี้คุณสามารถแชร์ผลลัพธ์ Customer Insights กับโปรแกรมประยุกต์ Microsoft Dataverse โดยใช้ Data Lake ที่จัดการ Microsoft Dataverse เมื่อคุณเชื่อมโยงสภาพแวดล้อม Dataverse ด้วย Customer Insights คุณจะได้รับตัวเลือกในการเปิดใช้งานการแบ่งปันข้อมูล
  สำหรับข้อมูลเพิ่มเติม ดู [จัดการการสภาพแวดล้อม](manage-environments.md)


## <a name="december-2020-updates"></a>การปรับปรุงประจำเดือนธันวาคม 2020

การปรับปรุงในเดือนธันวาคม 2020 ประกอบด้วยคุณลักษณะต่างๆ การอัปเกรดประสิทธิภาพ และการแก้ไขข้อบกพร่อง

### <a name="new-and-updated-features-in-december-2020"></a>คุณลักษณะใหม่และที่ปรับปรุงในเดือนธันวาคม 2020

#### <a name="data-enrichment"></a>การเพิ่มข้อมูล

- **ตราสินค้าที่ปรับปรุงและการเพิ่มความเกี่ยวข้องของความสนใจ**
  
  เราทำให้คะแนนความเกี่ยวข้องง่ายขึ้นเพื่อให้เข้าใจและใช้งานได้ง่ายขึ้น ตอนนี้คุณสามารถระบุลูกค้าได้อย่างรวดเร็วโดยพิจารณาจากความเกี่ยวข้องที่พวกเขามีต่อแบรนด์หรือความสนใจนั้น ๆ

  นอกจากนี้ เราได้เพิ่มตัวเลือกการกำหนดค่าใหม่เพื่อควบคุมวิธีที่คุณต้องการให้โปรไฟล์ลูกค้าของคุณสมบูรณ์ขึ้น 

  สำหรับข้อมูลเพิ่มเติม โปรดดู [เพิ่มโปรไฟล์ลูกค้าด้วยความเกี่ยวข้องของแบรนด์และความสนใจ](enrichment-microsoft-graph.md)

- **ควบคุมโปรไฟล์ที่จะเพิ่ม**

  ตอนนี้คุณสามารถเพิ่มโปรไฟล์ลูกค้าของคุณได้เพียงบางส่วนโดยมีตัวเลือกในการเลือกเอนทิตีเซ็กเมนต์แทนเอนทิตีลูกค้าเริ่มต้น สร้างเซ็กเมนต์ที่มีโปรไฟล์ลูกค้าที่คุณต้องการเพิ่มและเลือกในการกำหนดค่าการเพิ่มข้อมูลสำหรับชุดข้อมูลลูกค้าของคุณ
  ขณะนี้คุณลักษณะนี้มีให้เฉพาะสำหรับการเพิ่มข้อมูลโดย Experian และ HERE Technologies เราจะเปิดใช้ความสามารถนี้เพื่อเพิ่มข้อมูลให้มากขึ้นในไม่ช้า

  สำหรับข้อมูลเพิ่มเติม โปรดดู [เพิ่มโปรไฟล์ลูกค้าด้วยข้อมูลประชากรจาก Experian](enrichment-experian.md) หรือ [การเพิ่มข้อมูลโปรไฟล์ลูกค้าด้วย HERE Technologies](enrichment-here.md)

#### <a name="extensibility"></a>ความสามารถในการเพิ่ม

- **เปิดใช้งานเซ็กเมนต์ของคุณผ่าน Autopilot**

  ส่งออกเซ็กเมนต์ไปยัง Autopilot และใช้เพื่อวัตถุประสงค์ทางการตลาด สำหรับข้อมูลเพิ่มเติม โปรดดู [ตัวเชื่อมต่อสำหรับ Autopilot (ตัวอย่าง)](export-autopilot.md)

- **เปิดใช้งานเซ็กเมนต์ของคุณผ่าน SendGrid**

  ส่งออกเซ็กเมนต์ไปยัง SendGrid และใช้เพื่อวัตถุประสงค์ทางการตลาด สำหรับข้อมูลเพิ่มเติม โปรดดู [ตัวเชื่อมต่อสำหรับ SendGrid](export-sendgrid.md)

#### <a name="system-administration"></a>ผู้ดูแลระบบ

- **ประสบการณ์การจัดการสภาพแวดล้อมที่อัปเดต**
  
  ตอนนี้คุณสามารถสร้าง แก้ไข ลบ และรีเซ็ตสภาพแวดล้อมได้โดยตรงจากตัวเลือกสภาพแวดล้อมในส่วนหัวของแอป 
  
  นอกจากนี้ สภาพแวดล้อมที่คุณใช้จะถูกปักหมุดไว้ที่ด้านบนสุดของแผงสภาพแวดล้อม ดังนั้นคุณจึงไม่จำเป็นต้องค้นหาอีกต่อไป

  สำหรับข้อมูลเพิ่มเติม ดู [จัดการการสภาพแวดล้อม](manage-environments.md)

#### <a name="predictions"></a>การคาดคะเน

- **คาดคะเนการเลิกทำธุรกรรม**

  คุณลักษณะการคาดคะเนการเลิกทำธุรรกรรมช่วยให้คุณสามารถคาดคะเนความเป็นไปได้ที่ลูกค้าจะหยุดซื้อผลิตภัณฑ์หรือบริการได้โดยไม่ต้องอาศัยความช่วยเหลือจากนักวิทยาศาสตร์ข้อมูล  ด้วยการใช้คะแนนการคาดคะเน คุณสามารถรวมข้อมูลอื่นๆ เกี่ยวกับลูกค้าของคุณ เช่น คุณค่าของลูกค้า เพื่อสร้างเซ็กเมนต์ที่มีความเสี่ยงสูงหรือลูกค้าที่มีคุณค่าสูง ใช้เซ็กเมนต์นี้เพื่อกำหนดเป้าหมายลูกค้าโดยตรงผ่านกิจกรรมทางการตลาด การสนับสนุนลูกค้า และสถานการณ์อื่นๆ เพื่อลดความเสี่ยงในการเลิกใช้บริการ
 
  กำหนดค่าการกำหนดการเลิกใช้บริการเป็นกรอบเวลาตามเวลาที่เฉพาะเจาะจงสำหรับธุรกิจของคุณและกำหนดเวลาที่ลูกค้าจะได้รับการพิจารณาว่าเลิกใช้บริการ ตัวอย่างเช่น ร้านขายของชำอาจต้องการพิจารณาลูกค้าที่เลิกใช้บริการหากพวกเขาไม่ได้ซื้ออะไรเลยใน 30 วันที่ผ่านมา
 
  ขณะที่คุณสร้างการคาดคะเนต่อไป เราจะแนะนำคุณว่าข้อมูลใดที่จำเป็น และช่วยให้คุณสามารถแมปข้อมูลเกี่ยวกับธุรกิจของคุณกับฟิลด์ที่จำเป็นเพื่อคาดคะเนการเลิกใช้บริการของลูกค้า นอกจากนี้ คุณยังสามารถจัดกำหนดการในการฝึกโมเดลใหม่ตามข้อมูลใหม่ในระบบของคุณเพื่อปรับให้เข้ากับสถานการณ์ทางธุรกิจที่เปลี่ยนแปลงไป
 
  สำหรับข้อมูลเพิ่มเติม โปรดดู [การคาดคะเนการเลิกทำธุรกรรม (พรีวิว)](predict-transactional-churn.md)

  ภายในประสบการณ์นี้ คุณสามารถกำหนดค่าความหมายของการเลิกใช้บริการเป็นหน้าต่างตามเวลาเฉพาะสำหรับธุรกิจของคุณ ตัวอย่างเช่น ธุรกิจสตรีมมิ่งวิดีโอที่มีขั้นตอนการสมัครสมาชิกรายเดือนอาจต้องการให้ลูกค้าเลิกใช้บริการหลังจาก 15 วันหลังจากสิ้นสุดการสมัครใช้งาน

- **ค้นหาลูกค้าที่คล้ายกัน**
  
  ค้นหาลูกค้าที่คล้ายกันในฐานลูกค้าของคุณโดยใช้ปัญญาประดิษฐ์ โมเดลการเรียนรู้ของเครื่องของการจัดประเภทไบนารีมอบหมายคะแนนความคล้ายคลึงให้กับลูกค้าในเซ็กเมนต์ที่ขยาย คะแนนขึ้นอยู่กับความคล้ายคลึงกันกับลูกค้าในเซ็กเมนต์ต้นทาง ขึ้นอยู่กับคะแนนความคล้ายคลึงกัน โปรไฟล์ลูกค้าจะถูกเพิ่มไปยังเซ็กเมนต์ที่สร้างขึ้นใหม่

  บางครั้งเรียกว่าการสร้างแบบจำลองที่เหมือนกันในการตลาดดิจิทัล ซึ่งใช้แบบจำลอง AI เพื่อช่วยในการค้นหาลูกค้าที่คล้ายคลึงกับเซ็กเมนต์ลูกค้าของคุณอีกเซ็กเมนต์หนึ่งโดยคำนึงถึงแอตทริบิวต์เพิ่มเติม ไม่เพียงแต่อนุญาตให้คุณเลือกแอตทริบิวต์เท่านั้น แต่ยังช่วยให้คุณระบุจำนวนลูกค้าสูงสุดที่ควรอยู่ในเซ็กเมนต์ใหม่นี้ จากนั้นโมเดล AI จะคำนวณคะแนนความคล้ายคลึงกันสำหรับลูกค้าแต่ละรายตามคุณลักษณะที่คุณเลือก และค้นหาลูกค้าที่มีคะแนนความคล้ายคลึงกันโดยเฉลี่ยสูงกว่า เซ็กเมนต์ผลลัพธ์จะรวมลูกค้าที่มีลักษณะคล้ายกับลูกค้าในเซ็กเมนต์เดิมของคุณ    
  สำหรับข้อมูลเพิ่มเติม ให้ดูที่ [กค้าที่คล้ายกัน](find-similar-customer-segments.md)

- **เซ็กเมนต์ที่ทับซ้อนกันและตัวสร้างความแตกต่าง**

  การทับซ้อนกันของเซ็กเมนต์ช่วยให้คุณเห็นจำนวนและลูกค้ารายใดที่เป็นส่วนหนึ่งของเซ็กเมนต์สองเซ็กเมนต์ขึ้นไป ตัวอย่างเช่น เซ็กเมนต์ที่มีผู้ใช้จ่ายสูงซ้อนทับกับเซ็กเมนต์ลูกค้าที่มีความพึงพอใจสูงอย่างไร หรือเซ็กเมนต์ลูกค้าที่เลิกใช้บริการซ้อนทับกับเซ็กเมนต์ลูกค้าที่มีความพึงพอใจต่ำอย่างไร นอกจากนี้ คุณสามารถวิเคราะห์การเปลี่ยนแปลงที่ทับซ้อนกันตามแอตทริบิวต์เพิ่มเติมที่คุณเลือก

  ผู้สร้างความแตกต่างของเซ็กเมนต์จะเปิดเผยสิ่งที่ทำให้เซ็กเมนต์หนึ่งแตกต่างจากลูกค้าที่เหลือของคุณหรือจากเซ็กเมนต์อื่น สิ่งที่คุณต้องทำ คือระบุเซ็กเมนต์ และระบบจะระบุแอตทริบิวต์ของโปรไฟล์ และมาตรการที่แยกความแตกต่างของเซ็กเมนต์ในฟอร์มของรายการตัวสร้างความแตกต่างที่จัดอันดับ ตั้งแต่ตัวสร้างความแตกต่างที่แข็งแกร่งที่สุดไปจนถึงจุดอ่อนที่สุด    
  สำหรับข้อมูลเกี่ยวกับ ดูที่ [ข้อมูลเชิงลึกของเซ็กเมนต์ (แสดงตัวอย่าง)](segment-insights.md)

- **อายุการใช้งานของเซ็กเมนต์**
  
  กำหนดกำหนดการเพื่อเปิดใช้งาน หรือปิดใช้งานเซ็กเมนต์    
  สำหรับข้อมูลเพิ่มเติม ดูที่ [จัดการเซ็กเมนต์ที่มีอยู่](segments.md#manage-existing-segments)

## <a name="may-2020-updates"></a>การปรับปรุงเดือนพฤษภาคม 2020

การปรับปรุงในเดือนพฤษภาคม 2020 ประกอบด้วยคุณลักษณะต่างๆ การอัปเกรดประสิทธิภาพ และการแก้ไขข้อบกพร่อง

### <a name="new-and-updated-features-in-may-2020"></a>คุณลักษณะใหม่และอัปเดต ในเดือนพฤษภาคม 2020

#### <a name="data-ingestion"></a>การรับข้อมูล

- **การนำเข้าข้อมูลตามเวลาจริง: มุมมองประวัติ**

  เมื่อใช้ API ของเรา เพื่อรับการอัปเดตตามเวลาจริง คุณสามารถดูประวัติการรวมได้มากถึง 30 วัน สำหรับการอัปเดตเหล่านี้ คุณสามารถเข้าถึงการรวมของการเรียก API ที่สำเร็จหรือล้มเหลวทั้งหมด รวมถึง ผลลัพธ์ ระบบต้นทาง และข้อมูลเมตาที่เป็นประโยชน์อื่นๆ    
  สำหรับข้อมูลเพิ่มเติม โปรดดู [การเพิ่มข้อมูลแบบในเวลาจริง](real-time-data-ingestion.md)

- **การนำเข้าข้อมูลตามเวลาจริง: อัปเดตโปรไฟล์**

  ส่วนขยายของการนำเข้าข้อมูลตามเวลาจริง ช่วยให้คุณเห็นการเปลี่ยนแปลงในฟิลด์โปรไฟล์ผู้ใช้เฉพาะ ภายในไม่กี่วินาที    
  นอกเหนือจากฟังก์ชันการทำงานแบบเรียลไทม์สำหรับกิจกรรมแล้ว ระบบยังรองรับการปรับปรุงเวลาแฝงต่ำในฟิลด์โปรไฟล์ด้วย การอัปเดตตามเวลาจริงสำหรับฟิลด์โปรไฟล์มีเวลาหมดอายุ และดังนั้นจึงไม่ใช่การแทนที่สำหรับการรีเฟรชตามกำหนดเวลา    
  สำหรับข้อมูลเพิ่มเติม โปรดดู [การเพิ่มข้อมูลแบบในเวลาจริง](real-time-data-ingestion.md)

#### <a name="extensibility"></a>ความสามารถในการเพิ่ม

- **อัปเดตไทม์ไลน์และการแบ่งหน้าใน Add-in การ์ดลูกค้า**

  ไทม์ไลน์ของโซลูชัน Add-in การ์ดลูกค้าตรงกับไทม์ไลน์ของกิจกรรม การแบ่งหน้าของไทม์ไลน์ดีขึ้น โดยแสดงกิจกรรมได้สูงสุด 50 รายการ ในคราวเดียว นอกจากนี้ ยังช่วยให้โหลดกิจกรรมเพิ่มเติมในเส้นเวลาด้วย    
  สำหรับข้อมูลเพิ่มเติม ดู [Add-in การ์ดลูกค้า](customer-card-add-in.md)

- **Power Automate ทริกเกอร์สำหรับการเปลี่ยนแปลงเซ็กเมนต์**

  ทริกเกอร์สำหรับ Power Automate กำหนดสิ่งที่คุณสามารถสร้างโฟลว์จาก ทริกเกอร์ที่เพิ่มเข้ามาใหม่ช่วยให้คุณกำหนดเกณฑ์สำหรับเซ็กเมนต์ ตัวอย่างเช่น คุณสามารถสร้างการแจ้งเตือนที่จะถูกส่งเมื่อผ่านเกณฑ์ที่กำหนด    
  สำหรับข้อมูลเพิ่มเติม ดูที่ [ตัวเชื่อมต่อ Power Automate](export-power-automate.md)

- **การสนับสนุนหลายรายการสำหรับรุ่นที่กำหนดเอง**

  กำหนดค่าเวิร์กโฟลว์สำหรับโมเดลที่กำหนดเอง ด้วยเว็บเซอร์วิสของผู้เช่า Azure Machine Learning รายอื่น คุณสามารถลงชื่อเข้าใช้ผู้เช่า Azure Machine Learning เมื่อสร้างเวิร์กโฟลว์ใหม่สำหรับโมเดลที่กำหนดเอง ความสามารถนี้เป็นส่วนเสริมของความสามารถที่มีอยู่ในการรวมเข้ากับบริการเว็บ Azure Machine Learning ที่กำหนดเองของคุณเอง    
  สำหรับข้อมูลเพิ่มเติม โปรดดู [โมเดล Machine Learning ที่กำหนดเอง](custom-models.md)

#### <a name="segments"></a>เซ็กเมนต์

- **เส้นทางเอนทิตีอัตโนมัติ**

  เมื่อสร้างเซ็กเมนต์ ผู้ใช้ต้องกำหนดพาธของเอนทิตี ความสามารถนี้ใช้ขั้นตอนแรกในการกำหนดนิยามพาธเอนทิตีโดยอัตโนมัติ เพื่อให้คุณสามารถมุ่งเน้นไปที่เกณฑ์การแบ่งกลุ่มที่คุณมี    
  หากเอนทิตีที่คุณต้องการแบ่งกลุ่มลูกค้าของคุณเกี่ยวข้องโดยตรงกับเอนทิตีลูกค้ารวม คุณจะไม่จำเป็นต้องกำหนดเส้นทางเอนทิตีอีกต่อไป อย่างไรก็ตาม หากมีเส้นทางเอนทิตีที่เป็นไปได้มากกว่าหนึ่งเส้นทาง คุณยังต้องกำหนดด้วยตนเอง

- **การกระทำในหลายเซ็กเมนต์**
  
  ผู้ใช้สามารถเลือกหลายเซ็กเมนต์ และดำเนินการกับพวกเขา เช่น การรีเฟรชเซ็กเมนต์ ด้วยการคลิกเพียงครั้งเดียว    

- **รีเฟรชเซ็กเมนต์**

  ผู้ใช้สามารถรีเฟรชเซ็กเมนต์เดียว หรือเลือกเฉพาะเซ็กเมนต์ ที่พวกเขาต้องการรีเฟรช    

  
- **การปรับปรุงเซ็กเมนต์ผสม**

  ผู้ใช้สามารถ สร้าง แก้ไข และลบ เซ็กเมนต์ที่ยึดตามเซ็กเมนต์อื่น ตัวอย่างเช่น เซ็กเมนต์ที่สร้างขึ้นในเซ็กเมนต์อื่น ที่สร้างขึ้นในเซ็กเมนต์ที่สาม    

- **หน้ารายการเซ็กเมนต์**

  การออกแบบใหม่ของหน้าเซ็กเมนต์ ใช้รูปแบบรายการที่ช่วยให้คุณเห็นเซ็กเมนต์มากขึ้นในครั้งเดียว มีการเพิ่มฟิลด์ค้นหา เพื่อค้นหาเซ็กเมนต์อย่างรวดเร็ว ขณะนี้ ผู้ใช้สามารถใช้การกระทำ เช่น การดาวน์โหลดหรือลบหลายเซ็กเมนต์พร้อมกัน มีการนำเทรนด์ประสบการณ์ใหม่มาใช้ เพื่อระบุการเปลี่ยนแปลงที่สำคัญในเซ็กเมนต์อย่างรวดเร็ว    
  สำหรับข้อมูลเพิ่มเติม ดู [สร้างและจัดการเซ็กเมนต์](segments.md)

#### <a name="system-administration"></a>ผู้ดูแลระบบ

- **Customer Insights พร้อมใช้งาน ใน Microsoft Dynamics 365 Online Government**

  ด้วยช่องทางที่เพิ่มมากขึ้นสำหรับการโต้ตอบ ข้อมูลพลเมืองจะกระจัดกระจายไปทั่วทั้งระบบจำนวนมากมาย ที่นำไปสู่ข้อมูลที่ถูกปิดกั้น และมุมมองที่กระจัดกระจายของข้อมูลเกี่ยวกับการโต้ตอบของพลเมือง หากไม่มีมุมมองที่สมบูรณ์เกี่ยวกับการโต้ตอบของพลเมืองแต่ละช่องทาง เป็นไปไม่ได้ที่รัฐบาลจะทำให้ทันสมัย Microsoft มุ่งมั่นที่จะสนับสนุนความต้องการด้านเทคโนโลยีของรัฐบาลในการติดตามความคาดหวังของประชาชน เพื่อรับประสบการณ์ที่สอดคล้องและตอบสนอง    
  ด้วย รีลีส 2020 เวฟ 1 Dynamics 365 Customer Insights จะมีให้สำหรับ Government Community Cloud (GCC) ซึ่งเป็นสภาพแวดล้อมที่สร้างขึ้นเพื่อตอบสนองความต้องการด้านการปฏิบัติตามข้อกำหนดที่สูงขึ้นของหน่วยงานรัฐบาลของสหรัฐอเมริกา หน่วยงานได้รับมุมมองที่เป็นเอกภาพของประชาชนและใช้ AI ที่สร้างไว้ล่วงหน้า เพื่อรับข้อมูลเชิงลึกที่ปรับปรุงการปฏิสัมพันธ์ เพิ่มขีดความสามารถของพนักงาน และเปลี่ยนแปลงชุมชน ในขณะที่ลดความซับซ้อนด้านไอที และปฏิบัติตามมาตรฐานความปลอดภัยของสหรัฐอเมริกา Dynamics 365 Government เป็นไปตามข้อกำหนดของโครงการ US Federal Risk and Authorization Management (FedRAMP) ทำให้หน่วยงานรัฐบาลกลางของสหรัฐอเมริกาได้รับประโยชน์จากการประหยัดต้นทุน และความปลอดภัยที่เข้มงวดของ Microsoft Cloud

#### <a name="activities"></a>กิจกรรม

- **แมปกิจกรรมเอนทิตีกับประเภทกิจกรรมมาตรฐาน**
  
  การกำหนดค่ากิจกรรมและการจัดเก็บข้อมูลปัจจุบัน ขึ้นอยู่กับการออกแบบคงที่ เพื่อดูพวกเขาในไทม์ไลน์ ความหมายเชิงความหมายของกิจกรรม ที่มีศักยภาพสำหรับกรณีการใช้งานหลายกรณี ในโมเดล AI ไม่ได้ใช้อย่างเต็มที่ในขณะนี้ เราวางแผนที่จะทำให้ไทม์ไลน์ของกิจกรรมมีชีวิตชีวามากขึ้น โดยขึ้นอยู่กับประเภทของกิจกรรม และความเข้าใจที่ดีขึ้นของกิจกรรม คุณลักษณะนี้ มีวัตถุประสงค์เพื่อระบุประเภทกิจกรรมตามที่กำหนดไว้ใน Common Data Model สำหรับกิจกรรมใดๆ ที่รับ
  สำหรับข้อมูลเพิ่มเติม ดู [กิจกรรมของลูกค้า](activities.md)

  การนำเข้าข้อมูลในแบบเรียลไทม์จะให้ข้อมูลทันทีสำหรับการใช้งานจนกว่าการรีเฟรชตามกำหนดเวลาในภายหลังจะดึงข้อมูลนี้จากแหล่งข้อมูล    
  สำหรับข้อมูลเพิ่มเติม โปรดดู [การเพิ่มข้อมูลแบบในเวลาจริง](real-time-data-ingestion.md)

- **ปรับปรุงการเตรียมข้อมูล**
  
  เรียนรู้เพิ่มเติมเกี่ยวกับข้อมูลที่ติดอยู่ในเอนทิตี ด้วยการสรุปข้อมูล คุณสามารถเข้าใจคุณลักษณะคุณภาพของข้อมูลที่สามารถช่วยการดำเนินการที่เหมาะสม    
  สำหรับข้อมูลเพิ่มเติม ให้ดูที่ [สำรวจข้อมูลเอนทิตี](entities.md#exploring-a-specific-entitys-data)

- **ข้อมูลการวิเคราะห์จาก Dynamics 365 ด้วย Common Data Service**
  
  Common Data Service สามารถใช้เป็นวิธีสร้างแหล่งข้อมูล ลูกค้า Dynamics 365 ที่มีอยู่ สามารถนำเข้าเอนทิตีการวิเคราะห์ จาก Common Data Service ไปยัง Customer Insights แหล่งข้อมูลเดียวสามารถใช้ที่เก็บข้อมูลที่มีการจัดการ Common Data Service เดียวกันพร้อมกันได้ในสภาพแวดล้อม Customer Insights    
  สำหรับข้อมูลเพิ่มเติม โปรดดู [เชื่อมต่อกับข้อมูล ในที่จัดเก็บข้อมูลที่มีการจัดการ Common Data Service](connect-common-data-service-lake.md)

#### <a name="extensibility"></a>ความสามารถในการเพิ่ม

- **ส่งออกไปยัง LiveRamp**

  เริ่มการใช้งานข้อมูลของคุณ ใน LiveRamp® เพื่อเชื่อมต่อกับแพลตฟอร์มมากกว่า 500 รายการ ผ่านระบบนิเวศน์ดิจิทัล สังคม และทีวี ใช้ข้อมูลของคุณใน LiveRamp เพื่อกำหนดเป้าหมาย ปราบปราม และปรับแต่งแคมเปญโฆษณาในแบบของคุณ    
  สำหรับข้อมูลเพิ่มเติม ดูที่ [LiveRamp&reg; ตัวเชื่อมต่อ](export-liveramp.md)

- **Customer Insights Teams Add-in**
  
  บอทให้ความสามารถในการค้นหาสำหรับโปรไฟล์ลูกค้าแบบรวม นี่จะแสดงการ์ดที่มีมากถึง 15 ฟิลด์ จากโปรไฟล์ลูกค้าที่ได้ รายการที่ตรงกันหลายรายการส่งคืนรายการผลลัพธ์ที่คุณสามารถเลือกโปรไฟล์ได้    
  สำหรับข้อมูลเพิ่มเติม โปรดดู [บอท Teams สำหรับ Customer Insights](export-teams-bot.md)

#### <a name="measures"></a>การวัด

- **หน้ารายการการวัด**
  
  การปรับปรุงหน้าการวัด รวมถึงการสนับสนุนการกระทำในการวัดครั้งเดียวและการวัดหลายครั้ง นอกจากนี้ คุณจะพบฟิลด์ค้นหา เพื่อค้นหาและติดตามการวัดอย่างรวดเร็ว    
  สำหรับข้อมูลเพิ่มเติม ดู [สร้างและจัดการเซ็กเมนต์](segments.md)

- **การปรับปรุงการวัดผสม**
  
  ผู้ใช้สามารถ สร้าง แก้ไข และลบ การวัดที่ยึดตามการวัดอื่น ตัวอย่างเช่น การวัดที่สร้างขึ้นในการวัดอื่น ที่สร้างขึ้นในการวัดที่สาม

#### <a name="segments"></a>เซ็กเมนต์

- **ตัวดำเนินการอื่น**
  
  ตัวดำเนินการในการตั้งค่า ช่วยให้การแบ่งส่วนสำหรับลูกค้าด้วยค่าสตริงที่เป็นไปได้หลายค่า ก่อนที่จะเพิ่มตัวดำเนินการนี้ คุณจะต้องสร้างเซ็กเมนต์ดังกล่าวด้วยเงื่อนไข OR หลายรายการ ตัวดำเนินการในการตั้งค่าช่วยให้คุณทำได้ ด้วยเงื่อนไขเดียว    
  สำหรับข้อมูลเพิ่มเติม ดู [สร้างและจัดการเซ็กเมนต์](segments.md)

#### <a name="system-administration"></a>ผู้ดูแลระบบ

- **คัดลอกการตั้งค่าการกำหนดค่าไปยังสภาพแวดล้อมใหม่**
  
  คัดลอกการกำหนดค่าของคุณจากสภาพแวดล้อมหนึ่งไปยังอีกสภาพแวดล้อม ในขณะที่สร้างสภาพแวดล้อมใหม่ คุณสามารถเลือกสภาพแวดล้อมที่มีอยู่ที่คุณต้องการคัดลอกการกำหนดค่าจาก ขณะนี้เราสนับสนุน แหล่งข้อมูล การรวมข้อมูล ความสัมพันธ์ การวัด และเซ็กเมนต์ที่จะคัดลอก ข้อมูลประจำตัวของแหล่งข้อมูล และข้อมูลจริง จะไม่ถูกคัดลอก    
  สำหรับข้อมูลเพิ่มเติม ดู [จัดการการสภาพแวดล้อม](manage-environments.md)
