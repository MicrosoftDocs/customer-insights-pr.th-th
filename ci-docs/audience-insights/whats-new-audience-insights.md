# <a name="whats-new-in-the-audience-insights-capability-of-dynamics-365-customer-insights"></a>มีอะไรใหม่ในความสามารถข้อมูลเชิงลึกกลุ่มเป้าหมายของ Dynamics 365 Customer Insights

[!INCLUDE [cc-data-platform-banner](../includes/cc-data-platform-banner.md)]

พวกเรารู้สึกตื่นเต้นที่จะประกาศการปรับปรุงใหม่ล่าสุดของเรา! บทความนี้สรุปคุณลักษณะตัวอย่างสำหรับสาธารณะ การปรับปรุงความพร้อมใช้งานทั่วไป และการอัปเดตคุณลักษณะ เมื่อต้องการดูแผนคุณลักษะในระยะยาว ให้ดูที่ [แผนการนำออกใช้ของ Dynamics 365 และ Power Platform](/dynamics365/release-plans/)

เราเปิดตัวการอัปเดตตามภูมิภาค ดังนั้น บางภูมิภาคอาจเห็นคุณลักษณะก่อนภูมิภาคอื่นๆ หากไม่ได้ระบุไว้แตกต่างกัน คุณไม่จำเป็นต้องดำเนินการใดๆ และเราจะอัปเดตแอปโดยอัตโนมัติ โดยไม่ต้องหยุดทำงาน

> [!TIP]
> เมื่อต้องการ ส่งและลงคะแนนให้กับ การร้องขอคุณลักษณะ และ คำแนะนำผลิตภัณฑ์ ไปที่ [พอร์ทัลแนวคิดเกี่ยวกับโปรแกรมประยุกต์ Dynamics 365](https://experience.dynamics.com/ideas/categories/?forum=79a8c474-4e35-e911-a971-000d3a4f3343&forumName=Dynamics%20365%20Customer%20Insights)

## <a name="june-2021-updates"></a>การปรับปรุงเดือนมิถุนายน 2021

การปรับปรุงในเดือนมิถุนายน 2021 ประกอบด้วยคุณลักษณะต่างๆ การอัปเกรดประสิทธิภาพ และการแก้ไขข้อบกพร่อง

### <a name="data-ingestion"></a>การนำเข้าข้อมูล

- **ปรับปรุงการอัปเดตความคืบหน้าการรวมข้อมูล** ตอนนี้คุณสามารถดูการอัปเดตสถานะแบบไดนามิกที่ละเอียดยิ่งขึ้นในขั้นตอน [กระบวนการรวมข้อมูล](data-unification.md) คุณลักษณะนี้ช่วยให้คุณสามารถติดตามความคืบหน้าโดยละเอียดเพื่อทำความเข้าใจโฟลว์กระบวนการและดำเนินการหากขั้นตอนใดต้องให้ความสนใจ

### <a name="extensibility"></a>ความสามารถในการเพิ่ม

- **ส่งออกเซ็กเมนต์และข้อมูลอื่นๆ ไปยัง Salesforce Marketing Cloud** เราได้ขยายปลายทางการส่งออกของเราเพื่อรวม [Salesforce Marketing Cloud](export-salesforce.md) ตอนนี้คุณสามารถส่งออกเซ็กเมนต์และข้อมูลประเภทอื่นๆ ไปยัง Salesforce Marketing Cloud ผ่านการส่งออก SFTP ที่มีแบรนด์ได้แล้ว การนำเข้าข้อมูลสามารถทำได้โดยอัตโนมัติใน Salesforce และใช้เพื่อสร้างแคมเปญการตลาดที่มีประสิทธิภาพมากขึ้น  
 
- **ส่งออกเซ็กเมนต์ไปยัง ActiveCampaign** เราได้ขยายปลายทางการส่งออกของเราเพื่อรวม [แคมเปญที่ใช้งานอยู่](export-active-campaign.md) ตอนนี้คุณสามารถส่งออกเซ็กเมนต์เพื่อสร้างแคมเปญ จัดทำการตลาดทางอีเมล และทำงานกับกลุ่มลูกค้าเฉพาะใน ActiveCampaign
 
- **ส่งออกเซ็กเมนต์ไปยัง Sendinblue** เราได้ขยายปลายทางการส่งออกของเราเพื่อรวม [Sendinblue](export-sendinblue.md) ตอนนี้คุณสามารถส่งออกเซ็กเมนต์เพื่อสร้างแคมเปญ จัดทำการตลาดทางอีเมล และทำงานกับกลุ่มลูกค้าเฉพาะด้วย Sendinblue
 
### <a name="ux-updates"></a>การปรับปรุง UX 

- **หน้าลูกค้าและหน้ารายละเอียดโปรไฟล์แบบใหม่และที่ปรับปรุง** เราได้ออกแบบหน้าลูกค้าและหน้ารายละเอียดโปรไฟล์แบบใหม่เพื่อมอบประสบการณ์การใช้งานที่ดีขึ้นและประสิทธิภาพที่ดียิ่งขึ้น การเปลี่ยนแปลงเหล่านี้ช่วยให้คุณดู จัดเรียง ค้นหา และกรองลูกค้าได้ ขณะนี้ตัวกรองแสดงเป็นแบบ URL เพื่อให้แชร์ผลการค้นหากับผู้ใช้รายอื่นได้อย่างราบรื่น ผลลัพธ์การค้นหายังสามารถบันทึกเป็นเซ็กเมนต์ได้ด้วย    
  ตอนนี้หน้ารายละเอียดสำหรับโปรไฟล์ลูกค้าจะจัดกลุ่มข้อมูลเป็นส่วนย่อยต่างๆ เช่น ข้อมูลพื้นฐานของบุคคล รหัส และแอตทริบิวต์ของโปรไฟล์อื่นๆ เพื่อให้อ่านง่ายขึ้น ส่วนอื่นๆ ในหน้ารายละเอียดโปรไฟล์มีการโต้ตอบกันมากขึ้น ตัวอย่างเช่น ตอนนี้ส่วนกิจกรรมอนุญาตให้กรองและเรียงลำดับได้


## <a name="may-2021-updates"></a>การปรับปรุงเดือนพฤษภาคม 2021

การปรับปรุงในเดือนพฤษภาคม 2021 ประกอบด้วยคุณลักษณะต่างๆ การอัปเกรดประสิทธิภาพ และการแก้ไขข้อบกพร่อง

### <a name="data-ingestion"></a>การนำเข้าข้อมูล

- **ดูหรือแก้ไขข้อมูลเมตาหรือคำจำกัดความของเอนทิตี เมื่อแนบข้อมูลจาก Azure Data Lake Storage ของคุณ** ขณะนี้คุณสามารถดูและแก้ไขข้อมูลเมตาหรือการกำหนดเอนทิตีในข้อมูลเชิงลึกของผู้ชม เมื่อแนบข้อมูลจากโฟลเดอร์ Common Data Model ใน Azure Data Lake Storage ของคุณ ความสามารถนี้ให้ข้อเสนอแนะแบบเรียลไทม์ การตรวจสอบแบบจำลอง และการตรวจสอบข้อผิดพลาด ซึ่งช่วยให้คุณสามารถแก้ไขทั้ง model.json และ manifest.json ได้อย่างราบรื่น

### <a name="extensibility"></a>ความสามารถในการเพิ่ม

- **ปรับปรุงการส่งออกเซ็กเมนต์ กำหนดการที่กำหนดเอง และการทำซ้ำแล้ว** ขณะนี้คุณสามารถ [ดูการส่งออกทั้งหมดสำหรับเซ็กเมนต์เฉพาะ](export-destinations.md#view-exports-and-export-details) ในรายการได้ มุมมองใหม่นี้ช่วยในการจัดการวิธีการใช้เซ็กเมนต์เฉพาะและปรับเปลี่ยนที่มีอยู่ หรือสร้างการส่งออกใหม่    
  คุณสามารถ [กำหนดกำหนดการรีเฟรชที่กำหนดเอง](export-destinations.md#schedule-and-run-exports) สำหรับการส่งออกแต่ละรายการ หรือการส่งออกหลายรายการต่อครั้ง จนถึงขณะนี้ การส่งออกทั้งหมดถูกเรียกใช้พร้อมกับการรีเฟรชระบบทุกครั้ง    
  แทนที่จะสร้างการส่งออกใหม่ตั้งแต่ต้น คุณสามารถเริ่มต้นโดยอิงจากการส่งออกที่มีอยู่เพื่อประหยัดเวลา

- **ส่งออกเซ็กเมนต์ไปยัง Microsoft Advertising** เราได้ขยายปลายทางการส่งออกของเราเพื่อรวม Microsoft Advertising สร้างผู้ชม Customer Match บน Microsoft Advertising ด้วยข้อมูลโปรไฟล์ลูกค้าแบบรวมของคุณ และใช้ผู้ชมเหล่านี้สําหรับแคมเปญการโฆษณาของคุณ สำหรับข้อมูลเพิ่มเติม ดูที่ [ส่งออกเซ็กเมนต์ไปยัง Microsoft Advertising](export-microsoft-advertising.md)

- **ส่งออกเซ็กเมนต์ไปยัง LinkedIn Ads** เราได้ขยายปลายทางการส่งออกของเราให้รวม LinkedIn Ads และช่วยให้คุณสามารถปลดล็อก Contact Targeting และ Company Targeting ผ่าน LinkedIn โดยการส่งออกข้อมูลโปรไฟล์ลูกค้าแบบรวมของคุณ สำหรับข้อมูลเพิ่มเติม ดูที่ [ส่งออกเซ็กเมนต์ไปยัง LinkedIn Ads](export-linkedin-ads.md)


- **ส่งออกเซ็กเมนต์ไปยัง Omnisend** เราได้ขยายปลายทางการส่งออกของเราเพื่อรวม Omnisend ใช้เซ็กเมนต์ที่สร้างขึ้นในข้อมูลเชิงลึกของผู้ชมเพื่อสร้างแคมเปญ จัดทำการตลาดทางอีเมล และใช้ปกลุ่มเฉพาะของลูกค้าด้วย Omnisend สำหรับข้อมูลเพิ่มเติม ดูที่ [ส่งออกเซ็กเมนต์ไปยัง Omnisend](export-omnisend.md)

### <a name="predictions"></a>การคาดคะเน

- **รายงานการใช้งานข้อมูลป้อนเข้า** รายงานการใช้งานข้อมูลป้อนเข้ามีมุมมองแบบรวมของข้อผิดพลาดและคำเตือนที่อาจสร้างขึ้นจากการคาดคะเนแบบสำเร็จรูปของคุณ นอกจากนี้ ยังให้คำแนะนำในการปรับปรุงประสิทธิภาพของแบบจำลอง    
  รายงานจะพร้อมใช้งานหลังจากแบบจำลองเสร็จสิ้นกระบวนการฝึกอบรม ซึ่งถูกสร้างขึ้นสำหรับแต่ละรุ่นโดยแยกกัน โดยไม่คำนึงว่าจะสำเร็จหรือไม่
  ในปัจจุบัน คุณลักษณะนี้ใช้ได้เฉพาะสำหรับโมเดลการเลิกทำธุรกรรม สำหรับข้อมูลเพิ่มเติม ดูที่ [รายงานการใช้งานข้อมูลป้อนเข้า](manage-predictions.md#input-data-usability-report)

### <a name="relationships"></a>ความสัมพันธ์

- **ตัวสร้างภาพความสัมพันธ์** มุมมองตัวสร้างภาพความสัมพันธ์ช่วยให้คุณเห็นความสัมพันธ์ที่มีอยู่ทั้งหมดระหว่างเอนทิตีและคาร์ดินาลลิตี้ ความสัมพันธ์ได้รับการจัดระเบียบเป็นกลุ่ม: สร้างโดยผู้ใช้ ระบบ และความสัมพันธ์ที่มีการสืบทอด นอกจากนี้ คุณยังสามารถส่งออกมุมมองเป็นรูปภาพได้อีกด้วย สำหรับข้อมูลเพิ่มเติม โปรดดูที่ [ดูความสัมพันธ์](relationships.md#view-relationships) 

## <a name="april-2021-updates"></a>อัปเดต เมษายน 2021

การอัปเดตในเดือนเมษายน 2021 ประกอบด้วยคุณลักษณะ การอัปเกรดประสิทธิภาพ และการแก้ไขข้อบกพร่องต่างๆ

### <a name="data-unification"></a>การรวมข้อมูล
 
- **ประสบการณ์การผสานขั้นสูงสำหรับการรวมข้อมูล**    
  
   ขณะนี้เรามีประสบการณ์ของผู้ใช้ขั้นสูงในการกำหนดค่าการผสานของกระบวนการรวมข้อมูล การเปลี่ยนแปลงรวมถึงการจัดลำดับที่ใช้งานง่ายของฟิลด์ที่ผสานและสถิติโดยละเอียดเกี่ยวกับฟิลด์แบบรวมและแบบเดี่ยว

- **การจัดลำดับเอนทิตีใหม่และกำหนดค่าเรกคอร์ดแหล่งที่มาทั้งหมดลงในเอนทิตีลูกค้า**  
      
   ตอนนี้คุณสามารถจัดลำดับใหม่และลบเอนทิตีออกจากแผนการรวมกลุ่มที่มีอยู่ในกระบวนการรวมข้อมูลได้ ซึ่งให้ความยืดหยุ่นในการจัดลำดับเอนทิตีใหม่ในกระบวนการจับคู่ตามความต้องการทางธุรกิจ นอกจากนี้ เรายังเปิดใช้งานรวมเรกคอร์ดที่ไม่ตรงกันทั้งหมดไว้ในเอนทิตี *ลูกค้า* สุดท้าย ซึ่งช่วยให้พวกเขาสามารถกำหนดนิยามชุดข้อมูลโปรไฟล์ลูกค้าของพวกเขาได้

### <a name="enrichments"></a>การเพิ่มข้อมูล

 - **การเพิ่มข้อมูลใหม่: ที่อยู่ที่ปรับปรุงแล้ว**    
  
   เรามีความยินดีที่จะแนะนำการเพิ่มข้อมูลใหม่เพื่อเพิ่มความสมบูรณ์ที่อยู่ในข้อมูลลูกค้าของคุณ ที่อยู่ในข้อมูลของคุณอาจไม่มีโครงสร้าง ไม่สมบูรณ์ หรือไม่ถูกต้อง คุณลักษณะนี้ใช้โมเดลของ Microsoft เพื่อกำหนดมาตรฐานและเพิ่มความสมบูรณ์ที่อยู่ของคุณลงในรูปแบบ Common Data Model เพื่อความแม่นยำและข้อมูลเชิงลึกที่ดีขึ้น
 
   สำหรับข้อมูลเพิ่มเติม โปรดดู [การเพิ่มข้อมูลของโปรไฟล์ลูกค้าด้วยที่อยู่ที่ปรับปรุงแล้ว](enrichment-enhanced-addresses.md)

- **ประสบการณ์การกำหนดค่าที่แนะนำสำหรับการเพิ่มข้อมูล**    
  
   เราได้ทบทวนประสบการณ์การกำหนดค่าสำหรับการเพิ่มข้อมูลด้วยประสบการณ์ที่เรียบง่ายและที่แนะนำ ตอนนี้คุณมีกระบวนการที่ชัดเจนทีละขั้นตอนสำหรับการสร้างและการแก้ไขการเพิ่มข้อมูล
 
   นอกจากนี้ เรายังแยกการกำหนดค่าของการเชื่อมต่อสำหรับการเพิ่มข้อมูลของบุคคลที่สาม เพื่อให้สามารถใช้การเชื่อมต่อเดียวกันโดยการเพิ่มข้อมูลหลายรายการ เฉพาะผู้ดูแลระบบเท่านั้นที่สามารถกำหนดค่าการเชื่อมต่อใหม่ แต่การเชื่อมต่อที่สร้างขึ้นจะพร้อมใช้งานสำหรับทั้งผู้ดูแลระบบและผู้สนับสนุน    

   สำหรับข้อมูลเพิ่มเติม ดู [ภาพรวมของการเชื่อมต่อ](connections.md)

- **การเพิ่มข้อมูลหลายรายการในชนิดเดียวกัน**    
  
   ตอนนี้เราอนุญาตให้ผู้ใช้สร้างและจัดการการเพิ่มข้อมูลหลายรายการในชนิดการเพิ่มข้อมูลเดียวกัน ตัวอย่างเช่น ตอนนี้คุณสามารถสร้างการเพิ่มข้อมูลที่อยู่สองรายการแยกกัน เพื่อการเพิ่มข้อมูลเซ็กเมนต์ลูกค้าที่แตกต่างกันสองเซ็กเมนต์ ข้อจำกัดมีผลกับจำนวนการเพิ่มข้อมูลชนิดเดียวกันที่สามารถสร้างได้ และแตกต่างกันไปโดยขึ้นอยู่กับชนิดการเพิ่มข้อมูล
  
   สำหรับข้อมูลเพิ่มเติม โปรดดู [การเพิ่มข้อมูลสำหรับโปรไฟล์ลูกค้า](enrichment-hub.md)

#### <a name="segments"></a>เซ็กเมนต์

- **การเรียนรู้เกี่ยวกับเครื่องที่ดำเนินการโดยเซ็กเมนต์ที่แนะนําเพื่อปรับปรุงเมตริก** มีวิธีใหม่ในการค้นหาและสร้างเซ็กเมนต์ ระบบใช้โมเดล AI เพื่อแนะนำเซ็กเมนต์ที่สามารถช่วยปรับปรุง KPI (การวัดผล) ที่คุณกำลังติดตามอยู่ เราแสดงขอบเขตของอิทธิพลของแอตทริบิวต์ที่คุณเลือกในการวัดหรือแอตทริบิวต์หลักอื่น ๆ ข้อมูลนี้ช่วยในการค้นหาเซ็กเมนต์ที่เป็นไปได้ที่นำเสนอโอกาส    
  สำหรับข้อมูลเพิ่มเติม โปรดดู [เซ็กเมนต์แนะนํา (ตัวอย่าง)](suggested-segments.md)

#### <a name="data-unification"></a>การรวมข้อมูล

- **ประสบการณ์การจับคู่ที่เพิ่มขึ้น** ในพื้นที่การรวมข้อมูล ประสบการณ์การจับคู่ได้รับการอัปเดต ช่วยให้คุณกำหนดค่าและดูกฎการจับคู่ รวมถึงสถิติโดยละเอียดเพื่ออธิบายเพิ่มเติมว่าการจับคู่ทำงานอย่างไร มีตัวเลือกในการปิดใช้กฎการจับคู่ ดังนั้นจึงไม่ทำงานอีกต่อไป ในขณะที่ยังคงการกำหนดค่า ลากและวางกฎการจับคู่ และอื่น ๆ
  สำหรับข้อมูลเพิ่มเติม โปรดดูที่ [จับคู่เอนทิตี](match-entities.md)

- **ผลลัพธ์การลบข้อมูลซ้ำซ้อนจากกระบวนการจับคู่พร้อมใช้งานในรูปแบบเอนทิตี** ขณะนี้ผลลัพธ์ของกระบวนการคัดลอกข้อมูลซ้ำซ้อนจากกระบวนการจับคู่ถูกเขียนลงในเอนทิตีแยกต่างหากสำหรับการวิเคราะห์เพิ่มเติม เอนทิตีนี้ประกอบด้วยฟิลด์ที่ใช้ในกระบวนการขจัดข้อมูลซ้ำซ้อนและเรกคอร์ดผู้ชนะและเรกคอร์ดสำรองที่เกี่ยวข้องซึ่งรวมเข้ากับเรกคอร์ดผู้ชนะ
  สำหรับข้อมูลเพิ่มเติม โปรดดู [ผลลัพธ์การขจัดข้อมูลซ้ำซ้อนเป็นเอนทิตี](match-entities.md#deduplication-output-as-an-entity)

#### <a name="system-administration"></a>ผู้ดูแลระบบ

- **แบ่งปันข้อมูลไปยัง Microsoft Dataverse อย่างราบรื่น** ตอนนี้คุณสามารถแชร์ผลลัพธ์ Customer Insights กับโปรแกรมประยุกต์ Microsoft Dataverse โดยใช้ที่จัดเก็บข้อมูลดิบที่มีการจัดการของ Microsoft Dataverse เมื่อคุณเชื่อมโยงสภาพแวดล้อม Dataverse ด้วย Customer Insights คุณจะได้รับตัวเลือกในการเปิดใช้งานการแบ่งปันข้อมูล
  สำหรับข้อมูลเพิ่มเติม ดู [จัดการการสภาพแวดล้อม](manage-environments.md)
